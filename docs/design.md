Create a HLD (High-Level Design) document for the digital twin project. The intent of the project is to create a platform-agnostic universal digital twin for embedded platforms. The digital twin should have two parts:
- Digital Twin Server - an embedded application running "on-site" (embedded platform such as STM32/ESP32/Raspberry Pi/etc.). It's responsible for controlling some physical process (driving a motor, switching lights, relay, etc.) and exposing an API to interact with it. It should expose the api via abstract communication entity (IPC Server), which can be implemented in various ways. The user of the framework can provide the actual implementation on the initialization step. Digital Twin Server should be able to run custom bysiness logic also provided by the user.This logic should be also platform agnostic, while the actual interraction with the hardware should be abstracted by the framework. On the initialization step, the user should be able to provide the implementation of the hardware interaction layer, which will be used by the business logic.
- Digital Twin Client - an application running on a laptop/cloud/etc. It should have the same exact structure as the Digital Twin Server, but its platform-level implementations instead of controlling the physical process, should communicate with the Digital Twin Server to control the physical process remotely. The communication between them should be done via an abstract communication entity (IPC Client), which can be implemented in various ways. The user of the framework can provide the actual implementation on the initialization step.
Requirements:
- The framework should be implemented in C++ in platform agnostic way.
- The code between Client and Server should be highly reusable, while the only difference between them should be initialization with different implementations of the communication entity/hardware interaction layer.
- The framework should have small footprint and should be suitable for embedded platforms with limited resources.
- Before proceeding to the design phase, research existing open source solutions and how they can be used in our project. Consider also the possibility of using of HB-MVP design pattern. What are pros/cons of it?
- The framework should have crear and understandable structure and API.
